# Plan 3: Rewrite DATEV route to produce LODAS INI format with CP1252/CRLF

## Goal

Rewrite the `GET /api/v1/reports/datev` handler in
`apps/api/src/routes/reports.ts` so the output is a DATEV LODAS ASCII-importable
`.txt` file with the three mandatory INI sections (`[Allgemein]`,
`[Satzbeschreibung]`, `[Bewegungsdaten]`), CP1252 encoding, CRLF line endings,
and uses the 4 configurable Lohnartennummern from `TenantConfig`. Also install
`iconv-lite` as a direct API dependency and create the Wave 0 test file
`apps/api/src/routes/__tests__/reports.test.ts` that asserts DATEV-01 /
DATEV-02 / DATEV-03 behaviors.

## Threat Model (ASVS L1)

| Threat | ASVS | Mitigation | Implemented |
|--------|------|------------|-------------|
| Access control: non-admin triggers payroll export | V4.1 | `preHandler: requireRole("ADMIN")` already on route (reports.ts line 268) — MUST remain unchanged | preserved in rewrite |
| Tenant isolation: employees from another tenant leak into export | V4.2 | `where: { tenantId: req.user.tenantId }` on employee query — MUST remain in rewrite | preserved in rewrite |
| Tampering: malicious Lohnartennummer injection via config that breaks LODAS or misroutes hours | V5.1 | 4 fields validated in `tenantConfigSchema` (Plan 2) via `min(1).max(9999).int()`; default fallback via null-coalescing if config row is missing | enforced upstream by Plan 2 + null-coalescing in this plan |
| Injection: employee name with `;` or newline breaks the semicolon-separated data row format | V5.3 Output Encoding | `datevLine()` only emits numeric / short-code fields (pn, tag, ausfall, lohnart, stunden, tage) — employee names are NEVER written to the file. This is already the case in the current code and is preserved. | preserved — names never enter the export |
| Info disclosure: PII in audit log | V8.2 | Existing `app.audit({ ... newValue: { type: "DATEV", year, month } })` logs only metadata — no PII. MUST remain unchanged. | preserved in rewrite |
| DoS: huge response body consumes memory | V1.11 | Response is bounded by employee count × ~10 lines × ~80 bytes. For 1000 employees this is ~800 KB — well within Fastify's default reply size. No streaming needed for Phase 4. | accepted risk |
