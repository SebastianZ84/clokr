## Integration Findings

### INTEG-01 — Missing `isTrackOnly` guard in `recalculate-snapshots.ts` ⚠️ Medium

**File:** `apps/api/src/utils/recalculate-snapshots.ts:176`
**Affects:** SCHED-03

When a TRACK_ONLY MONTHLY_HOURS employee has their schedule changed, `settings.ts:336` calls `recalculateSnapshots()`. The function iterates all MONTHLY snapshots and writes:
```
carryOver = runningCarryOver + balanceMinutes
```
…without checking `schedule.overtimeMode`. For TRACK_ONLY employees, this writes non-zero carryOver to all historical snapshots and then updates `OvertimeAccount` to a non-zero balance — directly contradicting SCHED-03's guarantee.

**Fix required:** Add `isTrackOnly` check before writing carryOver, matching the pattern in the other three computation sites:
```typescript
const effectiveCarryOver = isTrackOnly ? 0 : runningCarryOver + balanceMinutes;
```

### INTEG-02 — `workingDaysInRange` denominator inconsistency ℹ️ Low

**Files:** `apps/api/src/utils/recalculate-snapshots.ts:135-142`, `apps/api/src/plugins/auto-close-month.ts:262-269`
**Affects:** TENANT-01 (hire-date edge case only)

`updateOvertimeAccount` was updated (WR-04 fix) to use full calendar month bounds for the `workingDaysInRange` denominator. The other two sites still use `effectiveStart` (hire-date-clamped). For mid-month hires, this produces different `dailySollMin` values between the live balance and the snapshot, causing a one-time discrepancy in the first partial month.

**Fix:** Align `recalculate-snapshots.ts` and `auto-close-month.ts` to use `monthRangeUtc()` bounds instead of `effectiveStart` for the working-days loop when computing the holiday-deduction denominator.

### INTEG-03 — Missing `isPureTracking` guard in `recalculate-snapshots.ts` ℹ️ Info

**File:** `apps/api/src/utils/recalculate-snapshots.ts:157-172`
**Affects:** BUG-02 (documentation/consistency only)

Leave deduction is applied without a `isPureTracking` guard. Likely harmless at runtime (calcExpectedMinutesTz returns 0 for monthlyHours=null), but diverges from the documented pattern at the other three sites.
