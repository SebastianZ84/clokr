<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User click -> theme.set() | User interaction sets theme via store — validated against themes array |
| localStorage -> data-theme attribute | Stored theme value applied to HTML element |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-06 | Tampering | Theme dot onclick | accept | `theme.set(t.id)` only accepts values from the typed `themes` array. The store validates values. No injection possible — `t.id` is a compile-time string literal from a static array. |
| T-08-07 | Tampering | style attribute on theme-dot-inner | accept | `style="background-color: {t.color}"` where `t.color` comes from a hardcoded static array in theme.ts (e.g., '#80377B'). Not user-controllable. No CSS injection vector. |
</threat_model>
